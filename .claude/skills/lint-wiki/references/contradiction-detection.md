# 矛盾检测详解

## 1. 事实矛盾

检测同一事实的不同陈述。

### 示例: GPT-4 发布日期

```markdown
## 发现矛盾

### 实体: OpenAI

**页面**: [[entities/openai]]
- 陈述1: "GPT-4 发布于 2023 年 3 月" [[sources/article-1]]
- 陈述2: "GPT-4 发布于 2023 年 6 月" [[sources/article-2]]

**建议**: 核实准确日期，更新页面并标注正确来源。
```

### 检查方法

1. 扫描所有页面的 Frontmatter 中的 `source_count` 引用
2. 提取每个实体/概念的关键事实
3. 比较来自不同来源的声明
4. 标记日期、数字、人物等关键数据的矛盾

## 2. 关系矛盾

检测关系描述的矛盾。

```markdown
### 关系矛盾

**关系**: 订单服务 → 库存服务

**页面**: [[systems/order-service]]
- 描述: "订单服务同步调用库存服务"

**页面**: [[architectures/diagrams/order-flow]]
- 描述: "订单服务异步调用库存服务"

**建议**: 确认调用方式，统一描述。
```

### 检查方法

1. 收集所有双向链接的关系描述
2. 比较不同页面对同一关系的描述
3. 检查调用方式、依赖类型等是否一致

## 3. 时间线矛盾

检测时间线的矛盾。

```markdown
### 时间线矛盾

**事件**: Transformer 架构发布

**页面**: [[concepts/transformer]]
- 时间: "2017 年"

**页面**: [[sources/transformer-paper]]
- 时间: "2018 年"

**建议**: 核实准确时间，更新页面。
```

### 检查方法

1. 识别具有时间属性的事实（发布日期、创建时间等）
2. 收集所有对这一事实的陈述
3. 检查时间戳是否一致
4. 优先使用官方来源或更近期的信息

## 输出格式

```markdown
# 矛盾检测报告

| 类型 | 实体/关系 | 矛盾描述 | 严重程度 |
|------|---------|---------|---------|
| 事实 | OpenAI | GPT-4 发布日期 | 🔴 高 |
| 关系 | Order→Inventory | 调用方式 | 🟡 中 |
| 时间线 | Transformer | 发布时间 | 🟡 中 |
```
