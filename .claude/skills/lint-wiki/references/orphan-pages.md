# 孤儿页面检测

## 什么是孤儿页面

孤儿页面是指没有被其他页面链接（即没有入站链接）的页面。

## 检测方法

### 1. 扫描所有链接

```markdown
## 孤儿页面

以下页面没有被其他页面链接：

### 实体
- [[entities/company-x]] - 无入站链接
- [[entities/person-y]] - 无入站链接

### 概念
- [[concepts/idea-z]] - 无入站链接

**建议**: 
1. 检查这些页面是否应该存在
2. 在相关页面中添加链接
3. 或考虑删除不重要的页面
```

### 2. 入站链接统计

```markdown
## 入站链接统计

| 页面 | 入站链接数 | 状态 |
|-----|-----------|------|
| [[concepts/transformer]] | 15 | ✅ 健康 |
| [[entities/openai]] | 12 | ✅ 健康 |
| [[entities/company-x]] | 0 | ⚠️ 孤儿 |
| [[services/create-order]] | 1 | ⚠️ 低度链接 |
```

## 处理策略

### 评估页面价值

- **保留**: 核心概念、重要实体、系统组件
- **删除**: 过时信息、重复内容、不相关页面
- **添加链接**: 确实相关但未链接的页面

### 自动添加链接

1. 识别页面的相关类别
2. 在同类页面中添加"相关页面"部分
3. 在父级概念页面中添加"子页面"链接

## 孤儿页面分类

### 高价值孤儿（应添加链接）

- 核心系统/服务：应在 `overview.md` 中链接
- 重要数据模型：应在相关服务页面中链接

### 低价值孤儿（考虑删除）

- 临时笔记/草稿
- 重复的概念总结
- 超过 2 年的旧信息

## 输出示例

```markdown
# 孤儿页面报告

## 统计

- 总页面数: 245
- 孤儿页面数: 12
- 孤儿比例: 4.9%

## 孤儿页面列表

### 需要添加链接 (8 个)
- [[systems/legacy-service]] → 建议在 overview.md 中链接
- [[components/cache-layer]] → 建议在 [[systems/api-service]] 中链接

### 建议删除 (4 个)
- [[entities/company-x]] → 无相关性
- [[concepts/deprecated-idea]] → 已过时
```
