# 查询类型说明

## 事实查询

**特征**: 询问"是什么"、"定义"、"概念"

**示例**:
- "什么是 Transformer？"
- "GPT-4 的定义是什么？"
- "解释一下注意力机制"

**处理方式**:
1. 定位概念页面
2. 提取定义和核心特性
3. 补充相关实体和应用场景

**输出格式**: Markdown 页面，包含概述、核心特性、应用场景

---

## 关系查询

**特征**: 询问"关系"、"联系"、"交互"

**示例**:
- "订单服务和库存服务的关系是什么？"
- "GPT-4 和 Transformer 有什么联系？"
- "用户和订单的关系"

**处理方式**:
1. 定位相关实体/系统页面
2. 分析页面间的链接关系
3. 提取交互流程和依赖关系

**输出格式**: 
- 关系概述
- 交互流程图（Mermaid sequenceDiagram）
- 详细说明

---

## 对比查询

**特征**: 询问"比较"、"对比"、"差异"、"区别"

**示例**:
- "比较 GPT-4 和 Claude 3 的差异"
- "对比微服务和单体架构"
- "PostgreSQL 和 MySQL 的区别"

**处理方式**:
1. 定位所有对比对象
2. 提取各对象的属性
3. 识别共同维度
4. 对比差异

**输出格式**: 
- 对比表格
- 主要差异说明
- 使用场景建议

---

## 列表查询

**特征**: 询问"哪些"、"列出"、"包含"

**示例**:
- "订单系统包含哪些组件？"
- "列出所有微服务"
- "有哪些大语言模型？"

**处理方式**:
1. 定位父级页面（系统、概念）
2. 提取子项列表
3. 补充每个子项的简要说明

**输出格式**: 
- 列表（带链接）
- 每项简要说明
- 统计信息

---

## 流程查询

**特征**: 询问"流程"、"过程"、"步骤"、"如何"

**示例**:
- "下单的完整流程是什么？"
- "如何部署一个微服务？"
- "订单处理过程"

**处理方式**:
1. 定位流程相关页面
2. 提取步骤信息
3. 识别步骤间的依赖关系
4. 构建流程图

**输出格式**: 
- 流程图（Mermaid graph 或 sequenceDiagram）
- 步骤详细说明
- 相关组件链接

---

## 架构查询

**特征**: 询问"架构"、"结构"、"设计"、"展示"

**示例**:
- "展示订单系统的架构"
- "微服务架构是什么样的？"
- "系统结构图"

**处理方式**:
1. 定位系统页面
2. 提取组件和模块信息
3. 识别组件间的关系
4. 构建架构图

**输出格式**: 
- 架构图（Mermaid graph）
- 核心组件说明
- 数据流说明

---

## 分析查询

**特征**: 询问"分析"、"优缺点"、"评估"、"评价"

**示例**:
- "分析微服务架构的优缺点"
- "评估 PostgreSQL 的性能"
- "评价 GPT-4 的能力"

**处理方式**:
1. 定位相关页面
2. 提取特性信息
3. 分析优势和劣势
4. 提供评估结论

**输出格式**: 
- 分析框架
- 优势列表
- 劣势列表
- 结论和建议
